    Introduction

    Welcome to the Custom App API! You can use our API reference to implement your endpoints, which help to integrate your app with Mylenio.

    Add your custom app to My Lenio

    Firstly, add your app to MyLenio as a Custom APP

    1. Navigate to Settings -> Apps
    2. Go to Custom Apps section and click onAdd New
    3. Fill App configuration fields as you need
    4. To integrate your API, go to External API integration and check API managed by MyLenio

    Custom API fields

    Field Description
    Admin email This user will be skipped when resyncing resources in MyLenio.
    Endpoint Basic URL of your API. i.e. https://my.custom.api.com/api
    Members link External link to access your user list. It is supposed authenticated and managed by your app
    User Creation strategy Should be set as Direct Creation instead you want to support invitations

    Authentication

    The API has all calls authenticated through user and password(Token). These values are included in the headers.

    The first thing you need to do is get the credentials to make authenticated calls to the API. For this, in MyLenio, Access your custom application, go to the API configuration section and copy the "API User" and "APi Password" fields.

    To authorize incoming requests, validate these headers below:

       //Authentication values in header
   {
    "X-AdminUser-Email": "api.user...@mylenio.com",
    "X-AdminUser-Token" : "..."
   }

    X-AdminUser-Email: user

    X-AdminUser-Token: token

    You must get email and token values from the Custom App you previously created in MyLenio.

    Basic

    API Version

    The URLs are generally formed as follows

    https://my.custom.api.com/api/[version]/[path]

    Payload Parameters

    Parameter value
    version v1
    path {request_path}

    Headers

    All request use application/json as content type

    Basic header to make requests:

       {
    "Content-type" : "application/json",
    ...
   }

    Error Handling

    See Errors section.

    Paginate Results

    Paginating your results will be optional. In case you want to paginate your responses, you need to include a links field in the root of the JSON object

    A sample response with pages

    {
 "data": [...],
 "links":
  {
   "self":"https://my.custom.api.com/api/v1/resources?page%5Bnumber%5D=2&page%5Bsize%5D=25",
   "first":"https://my.custom.api.com/api/api/v1/resources?page%5Bnumber%5D=1&page%5Bsize%5D=25",
   "prev":"https://my.custom.api.com/api/api/v1/resources?page%5Bnumber%5D=1&page%5Bsize%5D=25",
   "next": null,
   "last":"https://my.custom.api.com/api/api/v1/resources?page%5Bnumber%5D=2&page%5Bsize%5D=25"}
  }

    Invitations

    You might deal with invitations when creating or removing users from a workspace whether you want to integrate Mylenio with your application or want to build an intermediate app to integrate with another SaaS

    The simplest way is to have a direct creation user strategy. This means MyLenio will identify all users as Active.

    There are two strategies to support invitations

    • Invited Status - Using an additional field status in user model
    • With Invitations Using a dedicated endpoint for invitations
    When using Direct creation strategy invitation endpoints and user status fields are not needed

    Invited Status

    Add and additional field to your response in List Users

    List Users (Invited Status) - Response body

      {
    "data": [
      {
        "id": 1,
        "email": "user1@example.com",
        "name": "User 1",
        "status": "invited"
      },      {
        "id": 2,
        "email": "user2@example.com",
        "name": "User 2",
        "status": "active"
      }
    ]
  }
    It it possible to define as many statuses as you need. i.e "invited" , "active" , "other". However, the most important one for Mylenio is active.

    Resources

    Define a category

    Resource categories are the objects on which permissions will be assigned. i.e. Folder, repositories, boards, channels, etc.

    From MyLenio just need to follow the steps below:

    1. Access your custom app.
    2. Add a new category.
    3. Check API managed by Lenio field.
    4. Speficy the category endpoint field. i.e "/resources", "/boards", "/pages", etc.

    In your API implementation you must implement these groups of endpoints according to your category endpoint field.

    Endpoint Url Example with "/resources"
    Create/List /{category_endpoint} /resources
    Update /{category_endpoint}/{id}/update /resources/{id}/update
    Delete /{category_endpoint}/{id} /resources/{id}/
    Permission Roles /roles/{category_endpoint} /roles/resources
    List permissions /{category_endpoint}/{id}/permissions /resources/{id}/permissions
    Edit permissions /{category_endpoint}/{id}/permissions/{role_id} /resources/{id}/permissions/{role_id}
    The the following specifications will use "/resource" as category_endpoint
    Make sure of not using "/invitations" or "/users" as category_endpoint.

    Errors

    In order to simplify the error handling all Web API responses return CODE 200 and contain a JSON object. This object contains a top-level property data in most of cases.

    For failure results or invalid requests, data property is not present. Instead, the JSON object contains an error property set. This error property is an object with attributes code and message.

    A valid request returns a JSON structured like this:

    {
  "data": []
}

    A not valid request returns a JSON structured like this:

    {
  "error" : { "code" :  "not_found" , "message": "Requested Object not found"},
}
    In case of error, don't forget include the error property. Otherwise, MYLenio will determine that the request was valid.

    The Custom App API uses the following codes:

    Http Code Meaning
    200 Normal Response Code -- Both valid and not valid request.
    401 Unauthorized -- API credentials are wrong.
    5xx Other error -- An Unknown or unhandled error.
    4xx Other error -- An Unknown or unhandled internal error in MyLenio.
